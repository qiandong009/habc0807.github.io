前言

每次需求提测后，测试反馈过来的bug有些是重复性的，可能前几次的需求开发处理过，自测的时候忽视了。为了防止这种情况的出现，减少bug率，前端开发结束之后，可以自行翻看自测手册。

自测什么

  • 主功能
  • 兼容性和适配
  • 杂七杂八

主功能

每次需求都有一个需求文档，核对需求文档，自测主功能流程、页面业务和需求埋点。

  • 业务流程
  • 需求埋点
  • 核对设计稿
  • 分享
  • 生成的二维码能否识别；
  • 分享的图片、标题、文案是否一致；
  • 画图canvas，图片生成的是否完整；

兼容性和适配

移动端的设备众多，大小屏幕不一，很多时候只通过开发者工具调试，不能检测所有的真机问题。

在微信中的页面高度实际上要去除微信头和iphone机的返回bar所占高度。
iphone5/5s 按钮是否被遮挡

杂七杂八

页面滑动卡顿没有弹性

通过启动硬件加速特性解决

移动端弹出窗口后，底层还能滑动问题

解决这个问题的思路是：弹窗后禁用body滚动，关闭弹窗后释放body滚动。

调起软键盘，输入框及按钮被遮挡

安卓按钮文案不能垂直居中

按钮文案字体大小的奇偶数，和按钮的高度、行高的奇偶要保持一致，同奇同偶这样可以避免按钮内文案不能垂直居中问题。

取消 iOS 的内阴影

input的 placeholder 垂直居中

input::placeholder{
color: #ccc;
font-size: 15px;
line-height: normal;
}
